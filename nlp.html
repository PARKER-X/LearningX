Punishing Common Words

Part 5: Inverse Document Frequency (IDF)

Punishing Common Words

A word is important if it appears in this document but NOT in every document.

Definition

IDF(word) = log(
  Total number of documents
  /
  Number of documents containing the word
)

Example

We have 3 documents total.

Word Docs Containing It IDF

learning 3 log(3/3) = 0
love 2 log(3/2) ≈ 0.18

deep 1 log(3/1) ≈ 1.10

TF-IDF in Python

from sklearn.feature_extraction.text 
import TfidfVectorizer

docs = [
    "I love machine learning",
    "I love deep learning",
    "I hate machine learning"
]

vectorizer = TfidfVectorizer()
tfidf_matrix = vectorizer.fit_transform(docs)

print(vectorizer.get_feature_names_out())
print(tfidf_matrix.toarray())

Output: vectors showing importance of words ✅

Intuition Recap

  • TF = importance in this document
  • IDF = penalize words that appear everywhere
  • TF × IDF = the words that truly define the document

Cosine Similarity

Measure the angle between vectors, not their length 
cosine_similarity(A,B) = (A · B) / (||A|| × ||B||)
  • 1 → identical meaning
  • 0 → unrelated
  • -1 → opposite meaning

Embeddings in Python

from sklearn.feature_extraction.text 
import TfidfVectorizer
from sklearn.metrics.pairwise 
import cosine_similarity

docs = ["I love machine learning", 
        "I love deep learning", 
        "I hate machine learning"]

# TF-IDF vectors
vectorizer = TfidfVectorizer()
tfidf_matrix = vectorizer.fit_transform(docs)

# Cosine similarity
sim_matrix = cosine_similarity(tfidf_matrix)
print(sim_matrix)

🔹 Shows how documents relate in vector space
🔹 Embeddings extend this idea to capture meaning
