Part 3: Vocabulary & IDs

Why models don’t work with strings:
Computers and ML models cannot directly process text like "I love neural networks". They need numbers as input—vectors, matrices, and tensors.

Vocabulary Mapping

First, we create a vocabulary: a list of all tokens (words/subwords/characters) the model can handle. Each token is assigned a unique integer ID. 

"I"        → 17
"love"     → 2031
"neural"   → 9124
"networks" → 4512

Once we have the mapping, we can turn any sentence into a sequence of IDs:

"I love neural networks" → [17, 2031, 9124, 4512]

Important Notes:

  • At this stage, there is ZERO meaning in the numbers.
  • The numbers are simply IDs—like barcodes for words.
  • This step bridges raw text → numerical processing.

Task Heads — Turning Representations Into Decisions

Before Task Heads

Each token has a contextual vector: shape (batch_size, seq_len, hidden_dim)

Example: "I love cats" → [h₁("I"), h₂("love"), h₃("cats")]
Each hᵢ encodes meaning, role, context, and relations to other words.

Core Principle of Task Heads

A task head is a small neural network that maps contextual vectors → outputs
  • Usually linear layers + softmax/sigmoid
  • Sometimes CRF or decoders
  • The intelligence is not in the head — it just reads the representation

Sentence-Level Classification

  • One label per sentence (e.g., sentiment, topic, intent)
  • Use a single vector for the sentence ([CLS] token or mean pooling)
  • Linear → Softmax predicts class probabilities
Example: "I love cats" → Positive

Sentence Representation Strategies

  • [CLS] Token (BERT-style): use h_[CLS]
  • Mean Pooling (GPT-style): sentence_vector = mean(h₁, h₂, h₃)

Sentence Head Architecture

sentence_vector → Linear(hidden_dim → num_labels) → Softmax

Example: h_sentence → [2.3, -1.1] → Softmax → [0.95, 0.05] → Positive

Token-Level Classification

  • One label per token (NER, POS tagging)
  • Use all token vectors: [h₁, h₂, ..., hₙ]
  • Linear → Softmax per token
  • Optional CRF to enforce valid label transitions
Example: "Barack Obama was born in Hawaii" → [PER, PER, O, O, O, LOC]

Token Head Architecture

hᵢ → Linear(hidden_dim → num_tags) → Softmax

Output shape: (batch, seq_len, num_tags)

Sequence-to-Sequence Tasks

  • Input sequence → output sequence (translation, summarization)
  • Encoder: maps input → contextual meaning
  • Decoder: predicts next token using encoder context
  • Head = Linear(hidden_dim → vocab_size) → Softmax
Example: English → French translation

Language Modeling Head (GPT)

  • Predict the next token in a sequence
  • Head architecture: h_last → Linear(hidden_dim → vocab_size) → Softmax
Example: "I love neural" → predict "networks"

Multi-Task Heads

  • One backbone → multiple heads
  • Sentiment, NER, QA, retrieval heads share the same transformer
  • Task-specific linear layers read the same contextual vectors differently
Modern NLP models are powerful because they reuse context efficiently.

Key Takeaway

Task heads do not “understand” — they simply read learned representations and map them to the correct output.

spaCy NLP Pipeline Example

Code:

import spacy
from textblob import TextBlob

# Load English model
nlp = spacy.load("en_core_web_sm")

texts = [
    "Barack Obama was born in Hawaii.",
    "I love cats and programming."
]

for doc in nlp.pipe(texts):
    print(f"Text: {doc.text}")
    
    # POS tagging
    for token in doc:
        print(f"{token.text} → {token.pos_}")
    
    # Named Entities
    for ent in doc.ents:
        print(f"{ent.text} → {ent.label_}")
    
    # Dependency Parsing
    for token in doc:
        print(f"{token.text} → {token.dep_} → Head: {token.head.text}")
    
    # Lemmatization
    for token in doc:
        print(f"{token.text} → Lemma: {token.lemma_}")
    
    # Sentiment Analysis
    blob = TextBlob(doc.text)
    print(f"Polarity: {blob.sentiment.polarity}, Subjectivity: {blob.sentiment.subjectivity}")

Hugging Face Transformers Pipeline Example

Code:

from transformers import pipeline

texts = [
    "Barack Obama was born in Hawaii.",
    "I love cats and programming."
]

# Sentiment Analysis
sentiment_pipeline = pipeline("sentiment-analysis")
for text in texts:
    result = sentiment_pipeline(text)
    print(f"{text} → {result}")

# Named Entity Recognition (NER)
ner_pipeline = pipeline("ner", grouped_entities=True)
for text in texts:
    entities = ner_pipeline(text)
    print(f"{text} → {entities}")

# Question Answering
qa_pipeline = pipeline("question-answering")
context = "Barack Obama was born in Hawaii and was the 44th president of the USA."
question = "Where was Barack Obama born?"
answer = qa_pipeline(question=question, context=context)
print(f"Q: {question} → A: {answer['answer']}")

# Summarization
summarizer = pipeline("summarization")
article = "Hugging Face Transformers provide state-of-the-art models for NLP tasks..."
summary = summarizer(article, max_length=50, min_length=25, do_sample=False)
print(summary[0]['summary_text'])
